The CLI doesn't care about the prefix format as long as files sort lexicographically — both work. We pick `0001_` because it matches what every other story spec refers to.

**Reminder 4 — Scope: schema + types only.** Story 1.2 does NOT:

- Set Netlify env vars (Story 1.4)
- Touch auth flow / magic links (Story 1.11)

### `0001_init_schemas.sql` contract

```sql
-- 0001_init_schemas.sql
-- Create the non-public schemas referenced by the architecture:
--   admin         — founder admin tables (admin_users, affiliate_slots, etc.)
--   analytics_agg — aggregate-only metrics (affiliate_events_agg)
--
-- public is created by Postgres automatically; we don't touch it here.

CREATE SCHEMA IF NOT EXISTS admin;
CREATE SCHEMA IF NOT EXISTS analytics_agg;

-- Defense in depth: revoke default schema-level privileges from anon and authenticated.
-- RLS on individual tables is the primary gate; this is belt-and-suspenders so a
-- forgotten RLS toggle on a future admin table doesn't immediately leak.
REVOKE ALL ON SCHEMA admin FROM anon, authenticated;
REVOKE ALL ON SCHEMA analytics_agg FROM anon, authenticated;

-- service_role retains full access (Supabase managed-role convention).
-- postgres role retains ownership.
```

### `0002_profiles.sql` contract

```sql
-- 0002_profiles.sql
-- public.profiles — one row per authenticated user.
-- Owner: the user themselves (auth.uid() = user_id).
-- Created at onboarding (FR3), updated at settings (FR8).

-- Shared trigger function for updated_at maintenance.
-- Defined in public so future tables can reuse it; idempotent IF NOT EXISTS pattern.
CREATE OR REPLACE FUNCTION public.set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TABLE public.profiles (
  user_id            uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  cohort_tag         text,
  otr_or_local       text,
  default_state      text,
  dark_mode          boolean      NOT NULL DEFAULT true,
  analytics_opt_out  boolean      NOT NULL DEFAULT false,
  created_at         timestamptz  NOT NULL DEFAULT now(),
  updated_at         timestamptz  NOT NULL DEFAULT now(),

  CONSTRAINT profiles_otr_or_local_check
    CHECK (otr_or_local IS NULL OR otr_or_local IN ('otr', 'local')),
  CONSTRAINT profiles_default_state_check
    CHECK (default_state IS NULL OR default_state ~ '^[A-Z]{2}$')
);

-- Trigger: refresh updated_at on every UPDATE.
CREATE TRIGGER set_profiles_updated_at
  BEFORE UPDATE ON public.profiles
  FOR EACH ROW
  EXECUTE FUNCTION public.set_updated_at();

-- RLS: deny-by-default, then grant SELECT + UPDATE to the row owner.
-- INSERT policy is intentionally absent in Story 1.2; Stories 1.11/1.13 will add it
-- when the auth callback / onboarding flow first writes profile rows.
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY select_profiles_own
  ON public.profiles
  FOR SELECT
  TO authenticated
  USING (auth.uid() = user_id);

CREATE POLICY update_profiles_own
  ON public.profiles
  FOR UPDATE
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
```
