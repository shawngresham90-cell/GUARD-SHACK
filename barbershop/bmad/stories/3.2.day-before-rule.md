# Story 3.2: Day-before reminder rule

## Status
InProgress — implemented in backend/src/rules.ts + functions/scheduler/; unit tests pass (16/16).

## Story
**As a** shop owner,
**I want** each appointment booked for tomorrow to get one evening reminder,
**so that** clients remember without me lifting a finger.

## Acceptance Criteria
1. For `booked` appointments dated *tomorrow*, a reminder is sent once at/after
   `day_before_hour` (default 18:00 local).
2. `day_before_sent_at` is set in the same transaction as the send, so reruns
   never double-send.
3. The message renders `{name} {time} {service} {shop}` from settings.
4. Customers `opted_out` or not `ok_to_text` are skipped.
5. On send failure, `day_before_sent_at` is left null so the next run retries.

## Tasks / Subtasks

- [ ] Wire send + transactional timestamp update (AC: 2, 5)
- [ ] Render template (AC: 3)
- [ ] Apply consent/opt-out guard (AC: 4)

## Dev Notes
"Tomorrow" is computed in shop timezone. Depends on Stories 2.3, 3.1.

## Testing
- Unit: appointment tomorrow before 18:00 → no send; at/after → one send.
- Idempotency: run twice → exactly one send.
- Opt-out → skipped.
