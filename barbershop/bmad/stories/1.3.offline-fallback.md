# Story 1.3: Offline fallback

## Status
Draft

## Story
**As a** shop owner,
**I want** the app to keep working with no connection and reconcile when it comes
back,
**so that** cloud problems never block my day-to-day use.

## Acceptance Criteria
1. With the network off, all existing app features work against local storage.
2. Changes made offline are queued and synced on reconnect without duplicates.
3. On conflict, the most recent edit wins (last-write-wins by timestamp).
4. The app shows a subtle "offline / syncing / synced" indicator.

## Tasks / Subtasks
- [ ] Confirm local-first storage remains the source of truth (AC: 1)
- [ ] Implement an outbound sync queue with idempotent upserts (AC: 2)
- [ ] Add last-write-wins reconciliation (AC: 3)
- [ ] Add sync-status indicator (AC: 4)

## Dev Notes
The app is already a single-file offline PWA; this story ensures the new cloud
sync (1.2) degrades gracefully. Depends on Story 1.2.

## Testing
- Airplane-mode edits, reconnect, verify exactly one cloud row per entity.
- Simulate a conflicting edit on two devices; confirm newest wins.
