# Story 3.4: Quiet hours & dry-run

## Status
Draft

## Story
**As a** shop owner,
**I want** texts suppressed outside quiet hours and a preview mode before going
live,
**so that** I never text at a bad hour and I can trust the system first.

## Acceptance Criteria
1. No SMS is sent before `quiet_start` (08:00) or after `quiet_end` (21:00)
   local.
2. When `dry_run = true`, candidates are logged as `dry_run` in `message_log`
   with the exact body, and **no** Twilio call is made.
3. Switching `dry_run` to false enables live sending with no other changes.
4. Quiet-hours suppression does not consume the idempotency guard (a suppressed
   reminder can still send later in the allowed window).

## Tasks / Subtasks
- [ ] Add quiet-hours gate at the top of each run (AC: 1)
- [ ] Implement dry-run branch in the send path (AC: 2, 3)
- [ ] Ensure suppressed candidates keep `*_sent_at` null (AC: 4)

## Dev Notes
This is the safety gate for go-live: run dry for several days and review the
`dry_run` log before flipping `dry_run = false`. Depends on Stories 3.2, 3.3.

## Testing
- Unit: candidate at 22:00 → suppressed; same candidate at 08:30 → sent.
- Dry-run: confirm zero Twilio calls and a `dry_run` log row with correct body.
