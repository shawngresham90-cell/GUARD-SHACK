# Story 3.1: Cron worker skeleton

## Status
InProgress — implemented in backend/src/rules.ts + functions/scheduler/; unit tests pass (15/15).

## Story
**As a** shop owner,
**I want** a worker that wakes every 15 minutes and loads who might be due,
**so that** reminders can be evaluated without the app being open.

## Acceptance Criteria
1. A scheduled function runs every ~15 min independent of the app.
2. Each run loads `settings` and computes "now" in the shop timezone.
3. The run selects candidate appointments (booked, not yet sent for a type).
4. The run is observable via logs (counts of candidates considered).
5. With no candidates, the run exits cleanly and cheaply.

## Tasks / Subtasks
- [ ] Create `scheduler` Edge Function on a 15-min cron (AC: 1)
- [ ] Load settings + compute local now (AC: 2)
- [ ] Query candidate appointments (AC: 3)
- [ ] Add structured run logging (AC: 4, 5)
