# Story 2.1: Twilio account & number

## Status
Draft

## Story
**As a** shop owner,
**I want** a Twilio account I own with a sending phone number,
**so that** reminders can be delivered from a real number and billing is mine.

## Acceptance Criteria
1. A Twilio account is created and owned by the shop owner.
2. An SMS-capable phone number is provisioned.
3. Account SID and Auth Token are stored in Supabase secrets, never in git.
4. A manual test message can be sent from the number to the owner's phone.

## Tasks / Subtasks
- [ ] Create/verify Twilio account (owner-held billing) (AC: 1)
- [ ] Buy an SMS-capable local number (AC: 2)
- [ ] Store credentials in secrets (AC: 3)
- [ ] Send a manual test SMS to confirm the pipe works (AC: 4)

## Dev Notes
Number cost ~$1.15/mo per the spec. This story does not enable automatic
sending — it only proves the account/number. Live A2P sending requires Story
2.2.

## Testing
- Manual: send one test SMS via Twilio console/API and confirm receipt.
