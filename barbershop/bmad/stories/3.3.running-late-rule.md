# Story 3.3: Running-late reminder rule

## Status
InProgress — implemented in backend/src/rules.ts + functions/scheduler/; unit tests pass (15/15).

## Story
**As a** shop owner,
**I want** a "still coming?" text when an appointment is past due,
**so that** I can recover no-shows without watching the clock.

## Acceptance Criteria
1. For `booked` appointments where `now > start_at + late_threshold_min`
   (default 10) a late reminder is sent once.
2. `late_sent_at` is set in the same transaction as the send (no double-send).
3. The message renders the configured late template tokens.
4. `opted_out` / non-`ok_to_text` customers are skipped.
5. On failure, `late_sent_at` stays null for retry.

## Tasks / Subtasks
- [ ] Implement late decision in `rules.ts` (AC: 1)
- [ ] Wire send + transactional `late_sent_at` (AC: 2, 5)
- [ ] Render late template (AC: 3)
- [ ] Consent/opt-out guard (AC: 4)
