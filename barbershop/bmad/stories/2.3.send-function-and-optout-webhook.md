# Story 2.3: Send function & opt-out webhook

## Status
Draft

## Story
**As a** shop owner,
**I want** an outbound send path and an inbound webhook that honors STOP/START,
**so that** texts go out and anyone who opts out is never messaged again.

## Acceptance Criteria
1. A send function sends one SMS via Twilio and returns the message SID.
2. Each send writes a `message_log` row (`sent`/`failed` + SID).
3. An inbound webhook receives Twilio replies and validates
   `X-Twilio-Signature`.
4. `STOP`/`STOP ALL`/`UNSUBSCRIBE` set `customer.opted_out = true`; `START`
   clears it.
5. The send function refuses to send to an `opted_out` or non-`ok_to_text`
   customer.

## Tasks / Subtasks
- [ ] Implement `sendSms()` using Twilio creds from secrets (AC: 1, 2)
- [ ] Implement `sms-webhook` Edge Function with signature validation (AC: 3)
- [ ] Parse STOP/START keywords; update `customer` (AC: 4)
- [ ] Guard in `sendSms()` for opt-out/consent (AC: 5)

## Dev Notes
Twilio auto-handles STOP at the carrier level, but we still persist `opted_out`
so our scheduler never selects them. Templates rendered via the shared
`renderTemplate()`. Depends on Stories 1.1, 2.1.

## Testing
- Reply `STOP` from a test phone → confirm `opted_out = true` and no further
  sends.
- Reply `START` → confirm cleared.
- Unsigned webhook request → rejected.
