# Story 1.1: Supabase project & schema

## Status
InProgress — migrations written (`../../backend/supabase/migrations/`); awaiting
a real Supabase project to apply them (`supabase db push`).

## Story
**As a** shop owner,
**I want** a cloud database with tables for customers, appointments, and a
message log,
**so that** a server can see who needs texting and record what was sent.

## Acceptance Criteria
1. A Supabase project exists; connection details live in secrets, not git.
2. Tables `customer`, `appointment`, `message_log`, and a single-row `settings`
   exist per `architecture.md`.
3. `appointment` has `day_before_sent_at` and `late_sent_at` (nullable) for
   idempotency.
4. Row-level security restricts all rows to the shop owner.
5. A migration file reproduces the schema from scratch.

## Tasks / Subtasks

- [ ] Write migration creating the four tables (AC: 2, 3)
